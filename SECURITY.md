# Security Policy

## Supported Versions

We actively monitor and provide security updates for the following versions of **ClientHub-Bot**:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.0  | :white_check_mark: |
| < v1.0.0| :x:                |

## Reporting a Vulnerability

We take the security of our automation tools and our clients' data very seriously. If you discover any security vulnerability or data exposure risk within the bot or the synchronization pipeline, please do not open a public GitHub issue. 

Instead, report the vulnerability responsibly by following these steps:

1. Send a detailed email to our core team at **teamaytech@gmail.com**.
2. Include a description of the issue, the potential impact, and clear steps (or a proof-of-concept script) to reproduce the vulnerability.

### Our Response Commitment

* **Acknowledgment:** We will acknowledge receipt of your vulnerability report within 48 hours.
* **Triage & Fix:** Our team will investigate the issue and work on a fix immediately. We will keep you updated throughout the process.
* **Disclosure:** A public disclosure or release note update will be published once the fix is deployed securely to production.
